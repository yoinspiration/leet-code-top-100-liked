# Path Sum III

路径总和 III

## 题目要求

给定二叉树根结点 `root` 与整数 `targetSum`，统计满足以下条件的路径**条数**：路径从上往下（父到子），路径和等于 `targetSum`；路径起点、终点不必是根或叶结点。

题目链接：[437. 路径总和 III](https://leetcode.cn/problems/path-sum-iii/description/)

## 解法：DFS + 前缀和 + 哈希表 [PrefixSum.cpp](PrefixSum.cpp)

从根 DFS 到每个结点时，维护根到当前结点的前缀和 `curr`。若某祖先处前缀和为 `curr - targetSum`，则祖先的子结点（沿树向下）到当前结点这一段路径和为 `targetSum`。用哈希表记录「当前竖链上」各前缀和出现次数；进入子结点前 `prefix[curr]++`，处理完左右子树后回溯 `prefix[curr]--`，避免兄弟子树误用本支前缀。

初始化 `prefix[0] = 1`，表示「空前缀」出现一次，用于统计从根开始、和恰好为 `targetSum` 的路径。

结点值与路径和可能超出 32 位范围，实现中用 `long long` 存前缀和与目标。

- **时间复杂度：O(n)**，每个结点访问一次，哈希均摊 O(1)。
- **空间复杂度：O(h)**，递归栈 O(h)；哈希表规模不超过路径长度，最坏 O(n)。

### 解法说明

与数组上「和为 K 的子数组」一题思想相同：把「根到当前」看成前缀，差分得到中间一段路径和；区别在于树须在回溯时撤销哈希计数，使表内只保留从根到当前 DFS 栈上那条路径的前缀。
