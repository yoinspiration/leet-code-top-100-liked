# Linked List Cycle II

环形链表 II

## 题目要求

给定一个链表的头节点 `head`，返回链表开始入环的第一个节点。  
如果链表无环，则返回 `null`。

**示例：**

```
输入：head = [3,2,0,-4], pos = 1
输出：返回索引为 1 的链表节点
解释：链表中有一个环，尾部连接到第二个节点。

输入：head = [1,2], pos = 0
输出：返回索引为 0 的链表节点

输入：head = [1], pos = -1
输出：null
```

> 题目中 `pos` 仅用于表示链表尾部连接的位置，实际函数参数中不包含 `pos`。

## 解法：Floyd 判圈 + 入环点定位 [Floyd.cpp](Floyd.cpp)

### 算法思路

**第一阶段：判断有无环并找到相遇点**

- `slow` 每次走一步
- `fast` 每次走两步
- 若无环，`fast` 会先到达 `nullptr`
- 若有环，`slow` 与 `fast` 会在环内相遇

**第二阶段：从头节点和相遇点同步前进**

- 令一个指针 `ptr` 从 `head` 出发
- 令另一个指针从第一次相遇点出发（可复用 `slow`）
- 两者每次都走一步
- 再次相遇的位置就是入环点

### 为什么第二次相遇是入环点

设：

- `a`：从头节点到入环点的距离
- `b`：从入环点到第一次相遇点的距离
- `c`：环的总长度

当快慢指针第一次相遇时：

- 慢指针走了 `a + b`
- 快指针走了 `2(a + b)`
- 二者路程差是环长度的整数倍：`2(a + b) - (a + b) = k * c`

所以：

`a + b = k * c`  
`a = k * c - b = (k - 1) * c + (c - b)`

这表示：从 `head` 走 `a` 步到入环点；从相遇点走 `a` 步，等价于先走 `c - b` 到入环点，再绕若干圈，最终也会落在入环点。  
因此两指针同速前进会在入环点相遇。

### 复杂度分析

- **时间复杂度：O(n)**
- **空间复杂度：O(1)**
