# Longest Consecutive Sequence

最长连续序列

## 题目要求

给定一个未排序的整数数组 `nums`，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

请你设计并实现时间复杂度为 **O(n)** 的算法解决此问题。

**示例：**

```
输入：nums = [100,4,200,1,3,2]
输出：4
解释：最长数字连续序列是 [1, 2, 3, 4]。它的长度为 4。
```

## 解法：哈希集合 [HashSet.cpp](HashSet.cpp)

### 算法思路

使用哈希集合存储所有数字，实现 O(1) 的查找。关键思路是：只有当 `num-1` 不存在时，才从 `num` 开始计算连续序列，这样可以避免重复计算，每个连续序列只从最小值开始计算一次。

1. 将所有数字存入哈希集合
2. 遍历集合中的每个数字
3. 如果 `num-1` 不存在，说明 `num` 是某个连续序列的起点
4. 从 `num` 开始向右扩展，查找连续的数字，计算序列长度
5. 更新最大长度

### 复杂度分析

- **时间复杂度：O(n)**
  - 虽然看起来有两层循环，但每个数字最多被访问两次（一次作为起点，一次作为其他序列的一部分）
  - 哈希集合的查找操作是 O(1)
- **空间复杂度：O(n)**
  - 哈希集合最多存储 n 个数字

### 解法评价

巧妙地避免了重复计算，通过只从序列起点开始计算，确保每个数字最多被访问两次，从而实现了 O(n) 的时间复杂度。
