# Kth Largest Element in an Array

## 解法一：最小堆 [MinHeap.cpp](MinHeap.cpp)

### 算法思路

维护一个大小为 `k` 的**最小堆**，遍历数组时将每个元素入堆，若堆大小超过 `k` 则弹出堆顶（最小值）。

遍历结束后堆顶即为第 `k` 大的元素——堆内恰好保留了最大的 `k` 个数。

```
nums=[3,2,1,5,6,4], k=2

入3: heap=[3]
入2: heap=[2,3]
入1: heap=[1,2,3] → size>2, 弹1 → heap=[2,3]
入5: heap=[2,3,5] → 弹2 → heap=[3,5]
入6: heap=[3,5,6] → 弹3 → heap=[5,6]
入4: heap=[4,5,6] → 弹4 → heap=[5,6]
堆顶=5 ✓
```

### 复杂度分析

- **时间复杂度：O(n log k)**，每次堆操作 O(log k)
- **空间复杂度：O(k)**，堆最多存 k 个元素

## 解法二：快速选择（三路划分） [QuickSelect.cpp](QuickSelect.cpp)

### 算法思路

使用**三路划分**（Dutch National Flag），将区间分成三段：`< pivot | == pivot | > pivot`，返回等于 pivot 的区间 `[lt, gt]`。

第 `k` 大 = 排序后下标为 `n - k` 的元素（0-indexed），以此为 `target`。

- `target < lt`：答案在左侧，`hi = lt - 1`
- `target > gt`：答案在右侧，`lo = gt + 1`
- `lt ≤ target ≤ gt`：target 落在等于 pivot 的区间内，直接返回

三路划分的关键优势：遇到大量重复元素时，等于 pivot 的部分一次性全部归位，避免了二路划分在重复值上的 O(n²) 退化。

### 复杂度分析

- **时间复杂度：O(n) 平均**，重复元素多时性能更优
- **空间复杂度：O(1)**，原地操作

---

## 两种解法对比

| | 最小堆 | 快速选择 |
|---|---|---|
| 时间 | O(n log k) | O(n) 平均 |
| 空间 | O(k) | O(1) |
| 适合场景 | 数据流 / k 远小于 n | 静态数组 / 追求最优均摊 |
