# First Missing Positive

缺失的第一个正数

## 题目要求

给你一个未排序的整数数组 `nums`，请你找出其中没有出现的最小的正整数。

**要求：** 时间复杂度 O(n)，空间复杂度 O(1)。

**示例：**

```
输入：nums = [1,2,0]
输出：3

输入：nums = [3,4,-1,1]
输出：2

输入：nums = [7,8,9,11,12]
输出：1
```

## 解法：原地哈希 [InPlaceHash.cpp](InPlaceHash.cpp)

### 算法思路

**关键观察：** 对于长度为 n 的数组，缺失的第一个正数一定在 `[1, n+1]` 范围内。
- 如果数组包含 1 到 n 的所有数，那么答案是 `n+1`
- 否则，答案是 `[1, n]` 中第一个缺失的数

**核心思想：** 将每个数字 x 放到它应该在的位置（索引 `x-1`），然后遍历找到第一个位置不匹配的数字。

**算法步骤：**

1. **原地哈希**：遍历数组，对于每个 `nums[i]`：
   - 如果 `nums[i]` 在 `[1, n]` 范围内，且它不在正确的位置
   - 将它交换到正确的位置 `nums[nums[i] - 1]`
   - 重复直到无法交换（数字不在范围内或已在正确位置）

2. **查找缺失**：遍历数组，找到第一个位置 `i`，使得 `nums[i] != i + 1`
   - 如果找到，返回 `i + 1`
   - 如果所有位置都匹配，返回 `n + 1`

**示例：**

```
输入：nums = [3, 4, -1, 1]

第一步（原地哈希）：
- i=0: nums[0]=3, 应该放在索引2，交换 → [−1, 4, 3, 1]
- i=0: nums[0]=−1, 不在[1,4]范围内，跳过
- i=1: nums[1]=4, 应该放在索引3，交换 → [−1, 1, 3, 4]
- i=1: nums[1]=1, 应该放在索引0，交换 → [1, −1, 3, 4]
- i=1: nums[1]=−1, 不在范围内，跳过
- i=2: nums[2]=3, 已在正确位置
- i=3: nums[3]=4, 已在正确位置

第二步（查找缺失）：
- i=0: nums[0]=1 ✓
- i=1: nums[1]=−1 ≠ 2 ✗ → 返回 2
```

### 复杂度分析

- **时间复杂度：O(n)**
  - 每个元素最多被交换一次到正确位置
  - 两次遍历数组
- **空间复杂度：O(1)**
  - 只使用了常数额外空间（原地操作）

### 解法评价

最优解法，时间复杂度 O(n)，空间复杂度 O(1)。通过原地哈希，将数组本身作为哈希表，避免了额外空间的使用。
