# Coin Change

零钱兑换

## 题目要求

给你一个整数数组 `coins`，代表不同面额的硬币；以及一个整数 `amount`，代表总金额。

计算并返回可以凑成总金额所需的**最少硬币个数**。如果没有任何一种硬币组合能组成总金额，返回 `-1`。每种硬币的数量是无限的。

**示例：**

```
输入：coins = [1,5,11], amount = 15
输出：3
解释：15 = 5 + 5 + 5

输入：coins = [2], amount = 3
输出：-1
```

## 解法：动态规划 [DynamicProgramming.cpp](DynamicProgramming.cpp)

### 算法思路

定义 `dp[i]` 为凑成金额 `i` 所需的最少硬币数，初始化 `dp[0] = 0`，其余为正无穷。

对于每个金额 `i`，枚举所有面额不超过 `i` 的硬币：

```
dp[i] = min(dp[i - coin] + 1)
```

本质上是一个**完全背包**问题：每种硬币可重复使用，求恰好装满背包的最少件数。若 `dp[amount]` 仍为正无穷，说明无法凑成，返回 `-1`。

### 复杂度分析

- **时间复杂度：O(amount × n)**
  - 外层枚举金额 amount 次，内层枚举 n 种硬币
- **空间复杂度：O(amount)**
  - dp 数组大小为 amount+1
