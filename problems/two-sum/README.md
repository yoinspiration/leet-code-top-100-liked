# Two Sum

## 解法一：暴力枚举 [BruteForce.cpp](BruteForce.cpp)

### 算法思路

双重循环暴力枚举所有可能的数对，找到两数之和等于 target 的索引

### 复杂度分析

- **时间复杂度：O(n²)**
  - 外层循环：i 从 0 到 n-2，执行约 n 次
  - 内层循环：j 从 i+1 到 n-1
  - 总执行次数为 (n-1) + (n-2) + ... + 1 = n(n-1)/2 = O(n²)
- **空间复杂度：O(1)**
  - 只使用了固定大小的变量（i, j 等），没有额外的数据结构

### 解法评价

虽然时间复杂度较高，但思路直接，代码简单。

## 解法二：哈希表 [HashTable.cpp](HashTable.cpp)

### 算法思路

使用 `unordered_map` 存储已遍历的元素值及其索引，一次遍历即可找到答案。对于每个元素，计算 `complement = target - nums[i]`，如果 complement 已在 map 中，说明找到了答案。

### 复杂度分析

- **时间复杂度：O(n)**
  - 只需一次遍历数组，每个元素处理时间为 O(1)
- **空间复杂度：O(n)**
  - 哈希表最多存储 n 个元素

### 解法评价

时间复杂度更优，适合处理大规模数据。

## 题目要求

**进阶：你可以想出一个时间复杂度小于 O(n²) 的算法吗？**

题目鼓励找出时间复杂度更好的算法。本文档提供了两种解法，其中哈希表解法的时间复杂度为 O(n)，满足进阶要求。
