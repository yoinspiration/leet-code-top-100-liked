# Sort List

排序链表

## 题目要求

给你链表的头结点 `head`，请将其按升序排列并返回排序后的链表。  
题目要求时间复杂度为 `O(n log n)`。

## 解法：归并排序（自顶向下） [MergeSort.cpp](MergeSort.cpp)

### 算法思路

链表不支持随机访问，不适合用快速排序等需要频繁按下标访问的方案；归并排序非常适合链表：

1. **分割**
   - 用快慢指针找中点，把链表断成左右两段。
2. **递归排序**
   - 分别对子链表递归调用 `sortList`。
3. **合并**
   - 用「合并两个有序链表」的方式线性合并两段有序链表。

### 关键细节

- 基本情况：`head == nullptr` 或 `head->next == nullptr` 时，天然有序，直接返回。
- 找中点时使用 `fast = head->next`，让 `slow` 停在中点左侧，便于 `slow->next = nullptr` 断链。
- 合并阶段用哑结点 `dummy` 简化头结点处理。

### 复杂度分析

- **时间复杂度：O(n log n)**  
  每层合并总成本 `O(n)`，递归深度 `O(log n)`。

- **空间复杂度：O(log n)**（递归栈）  
  链表节点重连本身是原地的，不额外分配数组。

> 题目里常说“常数级空间复杂度”，严格来说自顶向下递归会使用 `O(log n)` 栈空间；若要严格满足常数额外空间，通常使用自底向上的迭代归并。
