# Top K Frequent Elements

## 解法一：最小堆 [MinHeap.cpp](MinHeap.cpp)

### 算法思路

1. 用哈希表统计每个元素的频次
2. 维护大小为 `k` 的**最小堆**（按频次排序），遍历频次表时：
   - 将 `{频次, 值}` 入堆
   - 若堆大小超过 `k`，弹出堆顶（频次最低的）
3. 遍历结束后堆内即为前 k 高频元素

```
nums=[1,1,1,2,2,3], k=2   频次: {1:3, 2:2, 3:1}

入{3,1}: heap=[{3,1}]
入{2,2}: heap=[{2,2},{3,1}]
入{1,3}: heap=[{1,3},{3,1},{2,2}] → size>2, 弹{1,3} → heap=[{2,2},{3,1}]
结果: [2,1]
```

### 复杂度分析

- **时间复杂度：O(n log k)**，统计 O(n)，堆操作 O(n log k)
- **空间复杂度：O(n)**，哈希表 + 堆

## 解法二：桶排序 [BucketSort.cpp](BucketSort.cpp)

### 算法思路

频次最大为 `n`，以频次为下标建桶：`bucket[i]` 存放所有频次恰好为 `i` 的元素。

从高频桶往低频桶遍历，依次收集元素直到满 `k` 个。

```
nums=[1,1,1,2,2,3], k=2   频次: {1:3, 2:2, 3:1}

bucket[3]=[1]
bucket[2]=[2]
bucket[1]=[3]

从 i=3 开始收集: res=[1] → i=2: res=[1,2] → 满k个，停止
结果: [1,2]
```

### 复杂度分析

- **时间复杂度：O(n)**，统计 + 建桶 + 收集均为线性
- **空间复杂度：O(n)**，桶数组 + 哈希表

---

## 两种解法对比

| | 最小堆 | 桶排序 |
|---|---|---|
| 时间 | O(n log k) | O(n) |
| 空间 | O(n) | O(n) |
| 适合场景 | 数据流 / k 远小于 n | 静态数组 / 追求线性 |
