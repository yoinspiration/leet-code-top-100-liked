# 3Sum

三数之和

## 题目要求

给你一个整数数组 `nums`，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足 `i != j`、`i != k` 且 `j != k`，同时还满足 `nums[i] + nums[j] + nums[k] == 0`。

请你返回所有和为 `0` 且不重复的三元组。

**注意：** 答案中不可以包含重复的三元组。

**示例：**

```
输入：nums = [-1,0,1,2,-1,-4]
输出：[[-1,-1,2],[-1,0,1]]
解释：
nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0 。
nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0 。
不同的三元组是 [-1,0,1] 和 [-1,-1,2] 。
注意，输出的顺序和三元组的顺序并不重要。
```

## 解法一：排序 + 双指针 [TwoPointers.cpp](TwoPointers.cpp)

### 算法思路

将问题转化为两数之和：固定第一个数，使用双指针找剩余两个数。

1. **排序数组**：先对数组排序，便于使用双指针和去重
2. **固定第一个数**：遍历数组，固定第一个数 `nums[i]`
3. **跳过重复的第一个数**：避免重复的三元组
4. **双指针找剩余两个数**：
   - `left = i + 1`，`right = n - 1`
   - `target = -nums[i]`（需要 `nums[left] + nums[right] == target`）
   - 根据和的大小移动指针：
     - 如果 `sum == target`：找到一组解，加入结果，并跳过重复元素
     - 如果 `sum < target`：`left++`
     - 如果 `sum > target`：`right--`
5. **去重处理**：在找到解后，跳过重复的 `left` 和 `right`

### 复杂度分析

- **时间复杂度：O(n²)**
  - 排序：O(n log n)
  - 外层循环：O(n)
  - 双指针：O(n)
  - 总体：O(n²)
- **空间复杂度：O(1)**
  - 除了结果数组外，只使用了常数额外空间
  - 如果考虑结果数组，则为 O(k)，k 是结果的数量

### 解法评价

最优解法，时间复杂度 O(n²)，空间复杂度 O(1)。通过排序和双指针技巧，巧妙地解决了去重问题。

## 解法二：暴力枚举 [BruteForce.cpp](BruteForce.cpp)

### 算法思路

三重循环枚举所有可能的三元组，使用 `set` 去重。

### 复杂度分析

- **时间复杂度：O(n³)**
  - 三重循环枚举所有可能的三元组
  - 每次插入 set 的时间复杂度为 O(log n)（n 是已存在的三元组数量）
- **空间复杂度：O(k)**
  - 需要额外的 set 存储结果去重，k 是结果的数量

### 解法评价

**注意：暴力枚举方法不能通过本题（会超时）。** 由于时间复杂度为 O(n³)，LeetCode 的测试用例中包含大规模数据，会导致超时。思路简单但效率极低，仅作学习参考。实际提交时**必须使用排序+双指针优化版本**。
