# Reverse Nodes in k-Group

K 个一组翻转链表

## 题目要求

给你链表的头节点 `head`，每 `k` 个节点一组进行翻转，返回修改后的链表。  
`k` 是正整数，且小于或等于链表长度。  
如果节点总数不是 `k` 的整数倍，那么最后剩余的节点保持原有顺序。

你不能只是改变节点内部的值，而是需要实际进行节点交换。

**示例：**

```
输入：head = [1,2,3,4,5], k = 2
输出：[2,1,4,3,5]

输入：head = [1,2,3,4,5], k = 3
输出：[3,2,1,4,5]
```

## 解法：哑结点 + 分组反转 [Iterative.cpp](Iterative.cpp)

### 算法思路

**核心思想：** 每次只反转一个长度为 `k` 的闭区间，反转完成后接回原链表，然后继续处理下一组。

- 使用哑结点 `dummy`，令 `groupPrev` 指向当前组前一个节点。
- 每轮先从 `groupPrev` 出发往后找第 `k` 个节点 `kth`：
  - 找不到说明剩余不足 `k` 个，直接结束。
- 设 `groupNext = kth->next`，随后把 `[groupPrev->next, kth]` 原地反转：
  - 采用常见链表反转模板，初始 `prev = groupNext`，`curr = groupPrev->next`。
  - 直到 `curr == groupNext` 结束，反转后 `kth` 成为新组头。
- 重新连接：
  - `groupPrev->next = kth`
  - 原组头反转后变组尾，令 `groupPrev` 移到它，进入下一轮。

### 复杂度分析

- **时间复杂度：O(n)**，每个节点最多被访问常数次。
- **空间复杂度：O(1)**，只使用常数个指针变量。

### 解法评价

相比递归解法，迭代写法避免了递归栈开销，更适合链表较长时使用。
