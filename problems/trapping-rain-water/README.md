# Trapping Rain Water

接雨水

## 题目要求

给定 `n` 个非负整数表示每个宽度为 `1` 的柱子的高度图，计算按此排列的柱子，下雨之后能接多少雨水。

**示例：**

```
输入：height = [0,1,0,2,1,0,1,3,2,1,2,1]
输出：6
解释：上面是由数组 [0,1,0,2,1,0,1,3,2,1,2,1] 表示的高度图，在这种情况下，可以接 6 个单位的雨水（蓝色部分表示雨水）。
```

## 解法一：双指针 [TwoPointers.cpp](TwoPointers.cpp)

### 算法思路

使用双指针从两端向中间移动，关键在于理解：**当前位置能够接到的雨水量取决于它左右两边的最大高度的较小值**。

1. 初始化 `left = 0`，`right = n - 1`
2. 维护 `leftMax`（左边遇到的最大高度）和 `rightMax`（右边遇到的最大高度）
3. 比较 `height[left]` 和 `height[right]`：
   - 如果 `height[left] < height[right]`：处理左边
     - 如果 `height[left] >= leftMax`：更新 `leftMax`
     - 否则：能够接到的雨水 = `leftMax - height[left]`，累加到结果
     - `left++`
   - 否则：处理右边（类似逻辑）
4. 重复直到 `left >= right`

**为什么这样是对的？**
- 当 `height[left] < height[right]` 时，可以确定 `left` 位置的雨水高度由 `leftMax` 决定（因为右边已经有更高的 `height[right]` 作为屏障）
- 反之，`right` 位置的雨水高度由 `rightMax` 决定

### 复杂度分析

- **时间复杂度：O(n)**
  - 只需一次遍历数组
- **空间复杂度：O(1)**
  - 只使用了常数额外空间

### 解法评价

最优解法，时间复杂度 O(n)，空间复杂度 O(1)。通过双指针技巧，在遍历过程中动态维护左右两边的最大高度，空间效率最高。

## 解法二：动态规划 [DynamicProgramming.cpp](DynamicProgramming.cpp)

### 算法思路

对于每个位置 `i`，能够接到的雨水量 = `min(左边最大高度, 右边最大高度) - height[i]`。

1. 从左到右遍历，计算 `leftMax[i]`（位置 i 左边（包括 i）的最大高度）
2. 从右到左遍历，计算 `rightMax[i]`（位置 i 右边（包括 i）的最大高度）
3. 遍历每个位置，计算能够接到的雨水量并累加

### 复杂度分析

- **时间复杂度：O(n)**
  - 需要三次遍历数组
- **空间复杂度：O(n)**
  - 需要两个数组存储 `leftMax` 和 `rightMax`

### 解法评价

思路清晰直观，时间复杂度 O(n)，但需要 O(n) 的额外空间。相比双指针解法，更容易理解，但空间效率较低。
