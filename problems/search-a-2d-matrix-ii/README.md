# Search a 2D Matrix II

搜索二维矩阵 II

## 题目要求

编写一个高效的算法来搜索 `m x n` 矩阵 `matrix` 中的一个目标值 `target`。

该矩阵具有以下特性：
- 每行的元素从左到右升序排列
- 每列的元素从上到下升序排列

**示例：**

```
输入：matrix = [[1,4,7,11],[2,5,8,12],[3,6,9,16],[10,13,14,17]], target = 5
输出：true

输入：matrix = [[1,4,7,11],[2,5,8,12],[3,6,9,16],[10,13,14,17]], target = 3
输出：false
```

## 解法：从角落开始搜索 [SearchFromCorner.cpp](SearchFromCorner.cpp)

### 算法思路

**核心思想：** 利用矩阵的有序性，从**右上角**（或左下角）开始搜索，每次可以排除一行或一列。

**为什么从右上角开始？**

- 右上角是**当前行的最大值**，也是**当前列的最小值**
- 如果 `matrix[i][j] > target`：说明当前列的所有元素都大于 target，可以排除这一列（向左移动）
- 如果 `matrix[i][j] < target`：说明当前行的所有元素都小于 target，可以排除这一行（向下移动）

**算法步骤：**

1. 从右上角 `(0, cols-1)` 开始
2. 比较当前值与 target：
   - 如果相等 → 返回 `true`
   - 如果 `current > target` → 向左移动（`j--`），排除当前列
   - 如果 `current < target` → 向下移动（`i++`），排除当前行
3. 如果越界仍未找到 → 返回 `false`

**示例：**

```
矩阵：
[[1,  4,  7,  11],
 [2,  5,  8,  12],
 [3,  6,  9,  16],
 [10, 13, 14, 17]]

搜索 target = 5：

步骤1：从 (0,3) = 11 开始
- 11 > 5 → 向左移动 → (0,2) = 7

步骤2：当前位置 (0,2) = 7
- 7 > 5 → 向左移动 → (0,1) = 4

步骤3：当前位置 (0,1) = 4
- 4 < 5 → 向下移动 → (1,1) = 5

步骤4：当前位置 (1,1) = 5
- 5 == 5 → 找到！返回 true
```

### 复杂度分析

- **时间复杂度：O(m + n)**
  - 最多遍历 m 行和 n 列
  - 每次移动排除一行或一列
- **空间复杂度：O(1)**
  - 只使用了常数额外空间

### 解法评价

最优解法，时间复杂度 O(m + n)，空间复杂度 O(1)。通过利用矩阵的有序性，从角落开始搜索，每次可以排除一行或一列，大大提高了搜索效率。
