# Linked List Cycle

环形链表

## 题目要求

给你一个链表的头节点 `head`，判断链表中是否有环。

**示例：**

```
输入：head = [3,2,0,-4], pos = 1
输出：true
解释：链表中存在一个环，其尾部连接到第二个节点。

输入：head = [1,2], pos = 0
输出：true

输入：head = [1], pos = -1
输出：false
```

> 题目中 `pos` 仅用于表示链表尾部指针连接到的索引位置，在代码实现中你看不到 `pos` 参数。

## 解法：快慢指针（Floyd 判圈算法） [Floyd.cpp](Floyd.cpp)

### 算法思路

**核心思想：** 使用两个速度不同的指针在链表上「追逐」：

- `slow` 每次走一步
- `fast` 每次走两步

如果链表中存在环：
- `fast` 会在环内追上 `slow`（相遇）

如果链表中不存在环：
- `fast` 会先到达链表末尾（`nullptr`），循环结束

### 算法步骤

1. 初始化：`slow = head`, `fast = head`
2. 在循环中前进指针：
   - `slow = slow->next`
   - `fast = fast->next->next`
3. 每次移动后：
   - 如果 `slow == fast` → 存在环，返回 `true`
   - 如果 `fast` 或 `fast->next` 为 `nullptr` → 无环，返回 `false`

### 复杂度分析

- **时间复杂度：O(n)**  
  - 最坏情况下，fast 追上 slow 前遍历整个链表长度级别的节点

- **空间复杂度：O(1)**  
  - 只使用了两个指针变量

### 解法评价

相比使用哈希表记录访问过的节点（需要 O(n) 额外空间），快慢指针算法在时间复杂度相同的前提下，空间复杂度更优，为 O(1)，是判断链表是否有环的经典解法。
