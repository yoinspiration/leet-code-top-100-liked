# Subarray Sum Equals K

和为 K 的子数组

## 题目要求

给你一个整数数组 `nums` 和一个整数 `k`，请你统计并返回该数组中和为 `k` 的连续子数组的个数。

**示例：**

```
输入：nums = [1,1,1], k = 2
输出：2

输入：nums = [1,2,3], k = 3
输出：2
```

## 解法一：前缀和 + 哈希表 [PrefixSum.cpp](PrefixSum.cpp)

### 算法思路

利用前缀和的思想：如果子数组 `[i, j]` 的和为 `k`，那么 `prefixSum[j] - prefixSum[i-1] = k`，即 `prefixSum[i-1] = prefixSum[j] - k`。

1. 使用哈希表 `prefixSumCount` 记录每个前缀和出现的次数
2. 初始化：`prefixSumCount[0] = 1`（表示前缀和为 0 出现 1 次，对应空子数组）
3. 遍历数组，计算当前前缀和 `prefixSum`
4. 查找 `prefixSum - k` 是否在哈希表中：
   - 如果存在，说明存在以当前位置结尾、和为 `k` 的子数组
   - 个数等于 `prefixSumCount[prefixSum - k]`
5. 更新当前前缀和的计数

**关键点：**
- 使用哈希表记录前缀和及其出现次数，可以在 O(1) 时间内查找
- 初始化 `prefixSumCount[0] = 1` 是为了处理从数组开头开始的子数组

### 复杂度分析

- **时间复杂度：O(n)**
  - 只需一次遍历数组
  - 哈希表的查找和插入操作都是 O(1)
- **空间复杂度：O(n)**
  - 哈希表最多存储 n 个不同的前缀和

### 解法评价

最优解法，时间复杂度 O(n)，空间复杂度 O(n)。通过前缀和和哈希表的结合，将问题转化为查找问题，大大提高了效率。

## 解法二：暴力枚举 [BruteForce.cpp](BruteForce.cpp)

### 算法思路

双重循环枚举所有可能的连续子数组，计算每个子数组的和，统计和为 `k` 的子数组个数。

### 复杂度分析

- **时间复杂度：O(n²)**
  - 外层循环：O(n)
  - 内层循环：O(n)
  - 每个子数组的和计算：O(1)（可以累加计算）
- **空间复杂度：O(1)**
  - 只使用了常数额外空间

### 解法评价

**注意：暴力枚举方法可能无法通过本题（会超时）。** 由于时间复杂度为 O(n²)，LeetCode 的测试用例中包含大规模数据，可能会导致超时。思路简单但效率低，仅作学习参考。实际提交时**必须使用前缀和+哈希表优化版本**。
