# Palindrome Linked List

回文链表

## 题目要求

给你一个单链表的头节点 `head`，请你判断它是否为回文链表。

**示例：**

```
输入：head = [1,2,2,1]
输出：true

输入：head = [1,2]
输出：false
```

## 解法：快慢指针 + 反转后半部分 [TwoPointers.cpp](TwoPointers.cpp)

### 算法思路

**核心思想：** 利用快慢指针找到链表中点，然后反转后半部分，与前半部分逐一比较。

1. **使用快慢指针找到中点**
   - `slow` 每次走 1 步，`fast` 每次走 2 步
   - 当 `fast` 或 `fast->next` 为空时，`slow` 就在中点位置

2. **反转后半部分链表**
   - 从 `slow` 开始，使用迭代反转链表
   - 得到后半部分反转后的头结点 `prev`

3. **比较前半部分和后半部分**
   - 指针 `p1` 从 `head` 开始，指针 `p2` 从 `prev` 开始
   - 逐一比较 `p1->val` 和 `p2->val`
   - 如果所有对应节点值都相等，则是回文链表

4. **（可选）恢复链表结构**
   - 如果需要保持链表原样，可以再反转一次后半部分

### 复杂度分析

- **时间复杂度：O(n)**  
  - 一次遍历找到中点，一次遍历反转，一次遍历比较

- **空间复杂度：O(1)**  
  - 只使用了有限的指针变量

### 解法评价

相比将链表元素拷贝到数组再判断回文（需要 O(n) 额外空间），这种方法在时间复杂度相同的情况下，空间复杂度更优，为 O(1)。
