# Container With Most Water

盛最多水的容器

## 题目要求

给定一个长度为 `n` 的整数数组 `height`。有 `n` 条垂线，第 `i` 条线的两个端点是 `(i, 0)` 和 `(i, height[i])`。

找出其中的两条线，使得它们与 `x` 轴共同构成的容器可以容纳最多的水。

返回容器可以储存的最大水量。

**说明：** 你不能倾斜容器。

**示例：**

```
输入：height = [1,8,6,2,5,4,8,3,7]
输出：49
解释：图中垂直线代表输入数组 [1,8,6,2,5,4,8,3,7]。在此情况下，容器能够容纳水（表示为蓝色部分）的最大值为 49。
```

## 解法一：双指针 [TwoPointers.cpp](TwoPointers.cpp)

### 算法思路

使用双指针技巧，从数组两端向中间移动：

1. 初始化 `left = 0`，`right = n - 1`
2. 计算当前容器的面积：`面积 = (right - left) × min(height[left], height[right])`
3. **关键观察**：移动较矮的那一边
   - 容器的面积受限于较矮的边
   - 如果移动较高的边，容器的宽度会减小，但高度不可能增加（仍受限于较矮的边），所以面积不可能增大
   - 只有移动较矮的边，才有可能遇到更高的边，从而增加面积
4. 重复步骤 2-3，直到两指针相遇

### 复杂度分析

- **时间复杂度：O(n)**
  - 只需一次遍历，每个元素最多被访问一次
- **空间复杂度：O(1)**
  - 只使用了常数额外空间

### 解法评价

最优解法，时间复杂度 O(n)，空间复杂度 O(1)。通过贪心策略，每次移动较矮的边，确保不会错过最优解。

## 解法二：暴力枚举 [BruteForce.cpp](BruteForce.cpp)

### 算法思路

双重循环枚举所有可能的容器组合，计算每个容器的面积，取最大值。

### 复杂度分析

- **时间复杂度：O(n²)**
  - 需要枚举所有可能的容器组合
- **空间复杂度：O(1)**
  - 只使用了常数额外空间

### 解法评价

**注意：暴力枚举方法不能通过本题（会超时）。** 由于时间复杂度为 O(n²)，LeetCode 的测试用例中包含大规模数据，会导致超时。思路简单但效率低，仅作学习参考。实际提交时**必须使用双指针优化版本**。
