# Move Zeroes

移动零

## 题目要求

给定一个数组 `nums`，编写一个函数将所有 `0` 移动到数组的末尾，同时保持非零元素的相对顺序。

**注意**：必须在不复制数组的情况下原地对数组进行操作。

**示例：**

```
输入: nums = [0,1,0,3,12]
输出: [1,3,12,0,0]
```

**进阶：** 你能尽量减少完成的操作次数吗？

## 解法一：双指针 [TwoPointers.cpp](TwoPointers.cpp)

### 算法思路

使用双指针技巧，分两步完成移动：

**第一步：将所有非零元素移动到前面**
1. `slow` 指针：指向下一个非零元素应该放置的位置
2. `fast` 指针：遍历整个数组
3. 当 `fast` 遇到非零元素时，将其移动到 `slow` 位置（如果 `fast != slow`），然后 `slow` 后移
4. 遍历结束后，`slow` 之前的位置都是非零元素，且保持了相对顺序

**第二步：统一置零**
5. 将 `slow` 之后的所有位置都置为 0

这种两步法避免了在遍历过程中多次置零操作，减少了实际的操作次数。

### 复杂度分析

- **时间复杂度：O(n)**
  - 第一步遍历数组一次，第二步最多遍历一次（从 slow 到数组末尾）
- **空间复杂度：O(1)**
  - 只使用了常数额外空间

### 解法评价

最优解法，时间复杂度 O(n)，空间复杂度 O(1)，代码简洁高效。

## 解法二：暴力枚举 [BruteForce.cpp](BruteForce.cpp)

### 算法思路

从后往前遍历数组，遇到 0 就将后面的所有元素向前移动一位，然后将最后一个位置置为 0。

### 复杂度分析

- **时间复杂度：O(n²)**
  - 外层循环遍历数组：O(n)
  - 内层循环移动元素：最坏情况下 O(n)
  - 当数组中有多个 0 时，会重复移动元素，效率较低
- **空间复杂度：O(1)**
  - 只使用了常数额外空间

### 解法评价

**低效解法，仅作参考。** 思路直观但效率低，不适合处理大规模数据。建议使用双指针优化版本。
