# Sliding Window Maximum

滑动窗口最大值

## 题目要求

给你一个整数数组 `nums`，有一个大小为 `k` 的滑动窗口从数组的最左侧移动到数组的最右侧。你只可以看到在滑动窗口内的 `k` 个数字。滑动窗口每次只向右移动一位。

返回 **滑动窗口中的最大值**。

**示例：**

```
输入：nums = [1,3,-1,-3,5,3,6,7], k = 3
输出：[3,3,5,5,6,7]
解释：
滑动窗口的位置                最大值
---------------               -----
[1  3  -1] -3  5  3  6  7       3
 1 [3  -1  -3] 5  3  6  7       3
 1  3 [-1  -3  5] 3  6  7       5
 1  3  -1 [-3  5  3] 6  7       5
 1  3  -1  -3 [5  3  6] 7       6
 1  3  -1  -3  5 [3  6  7]      7
```

## 解法一：单调队列 [MonotonicQueue.cpp](MonotonicQueue.cpp)

### 算法思路

使用单调递减队列（双端队列）来维护滑动窗口中的最大值：

1. **双端队列 `dq`**：存储元素的索引，保证队列中的元素按值从大到小排列
2. **遍历数组**：
   - 移除窗口外的元素（索引 <= i - k）
   - 移除队列中所有小于当前元素的元素（因为它们在当前窗口内不可能是最大值）
   - 将当前元素的索引加入队列
   - 当窗口大小达到 `k` 时，队列头部就是当前窗口的最大值

**关键点：**
- 队列中存储的是索引，而不是值，这样可以判断元素是否在窗口内
- 队列保持单调递减，保证队列头部始终是窗口内的最大值
- 移除小于当前元素的元素，因为它们不可能成为后续窗口的最大值

### 复杂度分析

- **时间复杂度：O(n)**
  - 每个元素最多被加入和移除队列各一次
  - 虽然看起来有嵌套循环，但实际上每个元素最多被处理两次
- **空间复杂度：O(k)**
  - 队列最多存储 `k` 个元素（窗口大小）

### 解法评价

最优解法，时间复杂度 O(n)，空间复杂度 O(k)。通过单调队列巧妙地维护了滑动窗口的最大值，避免了重复计算。

## 解法二：暴力枚举 [BruteForce.cpp](BruteForce.cpp)

### 算法思路

枚举每个滑动窗口，在每个窗口内找到最大值。

### 复杂度分析

- **时间复杂度：O(n * k)**
  - 有 `n - k + 1` 个窗口
  - 每个窗口需要 O(k) 时间找到最大值
- **空间复杂度：O(1)**
  - 除了结果数组外，只使用了常数额外空间
  - 如果考虑结果数组，则为 O(n - k + 1)

### 解法评价

**注意：暴力枚举方法可能无法通过本题（会超时）。** 由于时间复杂度为 O(n * k)，LeetCode 的测试用例中包含大规模数据，可能会导致超时。思路简单但效率低，仅作学习参考。实际提交时**必须使用单调队列优化版本**。
